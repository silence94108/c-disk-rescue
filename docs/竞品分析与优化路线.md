## 0. 结论先行

- **市场不空白,但"可信"是空位。** 功能层面(清理/搬家/大文件)竞品早已同质化;没有一家同时做到「免费 + 无捆绑 + 操作透明可验证 + 全自助」。两家最直接的竞品都在 2026 年转向收费/会员制,免费阵地正在空出来。
- **搬家/迁移是腾空间的最大杠杆**(竞品用户实测:纯清理 1~8GB,迁移类功能 20~70GB),印证 F3 的战略地位;**其次是系统级深度清理**(WinSxS/休眠文件/还原点),这是本项目目前最大的功能差距。
- **竞品的真正壁垒是路径特征库(规则数据),不是技术。** symlink/junction、RM API、回收站删除全是公开机制,本项目已全部掌握。追赶特征库有正当且高效的方法(§5.2)。
- **调研纪律教训:软文不可信,必须回官网验原文。** 两家竞品都被网络评测描述为"完全免费",实际一家挂价 19.9 元/台、一家清理功能锁在会员墙后。

## 2. 竞品档案:磨针C盘清理

> 信源:官网原文(tools.nwumba.cn,2026-07-12 抓取)+ 官网 105 条用户评论。网络评测多为软文,不采信。

- **主体**:成都磨针科技有限公司,蜀ICP备2021029677,公安备案,2023-03 起运营。新版为 Flutter 桌面应用(报错 DLL 为 Flutter Windows 插件),约 79MB。
- **价格**:19.9 元/单台永久,**2026-07-11 起正式挂价**(此前长期以"开通会员"形式收费,网络"完全免费"评测均失实)。一个授权绑 2 台机器码。
- **商业模式**:软件引流 + 微信客服 + 远程人工兜底。官网对比表的假想敌是"某宝按次几十元的人工清理服务"。评论区大量"客服远程操作""维护小哥远程搞定"。
- **王牌功能"负载均衡"**:批量把 C 盘目录搬到其他盘 + symlink 回指,与本项目 F3 同技术路线。区别:它靠注销/重启绕开文件锁、敢动系统级目录(评论:"需要注销再点开始""权限是 system 要按教程改"),单次腾 20~70GB;有常驻组件持续搬新增数据("会员期间可维持 C 盘空间基本稳定")。
- **红旗**:官网原文建议"关闭杀毒软件,因为杀毒软件会阻拦安装、扫描和文件清理";清理内容不透明(用户评论:"不知道转移了什么文件,会不会影响之后的使用");2025-06 新版出现 DLL 缺失打不开、会员登录失败等质量事故;UI 被用户评价"有点 low"。
- **对本项目的启示**:付费意愿真实存在(研究生/设计师/AI 研究员等大型软件用户愿付费);迁移是空间杠杆之王;它靠人工客服掩盖自动化不足——本项目反其道:规则少而准、全自动可撤销。

## 3. 竞品档案:Windows优化大师 V7(莫停之科技)

> 信源:官网原文(youhua.imeitools.com,2026-07-12 抓取)+ 下载站会员分析。注意与 2000 年代经典 wopti「Windows优化大师」**不是同一产品线**,系珠海公司借壳老品牌。

- **主体**:珠海市莫停之科技有限公司(粤ICP备2022015466),关联"有用科技"(wyouhua.com),两站产品列表高度相似。产品矩阵:优化大师、驱动大师、DLL系统修复、打印机驱动修复大师、电脑维修大师(人工远程)、PDF转换器、"DS一键部署大师"(蹭 DeepSeek 热点)——典型马甲软件矩阵厂。
- **价格**:官网通篇"免费下载"但不标价;实际**垃圾清理/C盘瘦身等核心功能锁在会员墙后**(3个月/12个月/3年/终身档位),免费的只有硬件检测、跑分等。官网用户评价区自我暴露("充了终身会员""永久会员!")。
- **功能面**:大而全电脑管家路线——垃圾清理、C盘瘦身(含软件/大文件搬家)、弹窗拦截、开机加速、重复文件/相似图片、微信/QQ 专清、注册表清理、"智能自动优化"(空闲时自动清理 = 常驻后台)、万能工具箱(截图/OCR/硬件)。
- **红旗**:营销数字无法验证("每天清理 700TB""硬盘寿命提升 7.2 倍");官网评价模板化且混入其他产品好评;自家页面挂"精品软件推荐"位(自我捆绑);借壳命名蹭老品牌与竞品品牌(驱动大师≈驱动人生/驱动精灵)。
- **对本项目的启示**:它验证了"下载免费、功能收费"模式的用户反感点正是本项目的机会;其"智能自动优化"常驻后台正是需求文档 §1.3 红线明确禁止的方向,不跟。

## 5. 值得跟进的优化项

排序原则:腾空间杠杆 × 实现成本 × 风险。每项落地前须过一遍需求文档 §1.3 红线检查。

### 5.1 【P1】系统深度清理包

**依据**:磨针"一次腾几十 G"口碑的真正来源;本项目目前单机清理量级 1~8GB,差距主要在此。

**细目**(均为公开系统机制,按风险分级):

| 子项 | 机制 | 预期空间 | risk 分级 | 要点 |
|------|------|---------|----------|------|
| WinSxS 组件存储 | `DISM /Online /Cleanup-Image /StartComponentCleanup` | 2~10GB | cost | 系统 API 自行判定可清部分,安全;耗时长需进度反馈;needsAdmin |
| 休眠文件 hiberfil.sys | `powercfg /h off` | 内存×40%(数 GB) | cost | 代价:失去休眠+快速启动,explain 必须讲清;大文件页现有"不能删"提示可升级为「想腾这块?去关闭休眠」引导(scan.rs 的 deletability 已识别此文件) |
| 系统还原点 | `vssadmin delete shadows`(保留最新) | 0~10GB | caution | 只提供"删除旧还原点、保留最新",绝不全删;默认不勾选 |
| 系统内存转储 | `%WINDIR%\MEMORY.DMP`、`%WINDIR%\Minidump` | 0~数 GB | safe | 纯规则库新增,needsAdmin;与现有 crash-dumps(用户级 CrashDumps 目录)互补 |
| Windows.old | cleanmgr handler / DISM | 10~30GB | cost | 现有规则为"只展示跳过"(cleaner.rs:529-535),按该文档既定路径接入系统清理接口 |
| 旧驱动包 | pnputil 枚举孤儿包 | 数 GB | cost | **已有完整方案,见[融合方案文档 §1](./融合方案-驱动包与孤儿profile.md),不在此重复** |

**红线检查**:全部走系统 API 判定、不做文件级遍历删除;每项独立规则、独立分级、独立 explain;needsAdmin 项沿用现有权限矩阵。

**验收**:一台常规使用 2 年以上的 Win10/11 真机,深度清理包合计释放 ≥10GB,且重启后系统与常用软件功能正常。

### 5.2 【P1】特征库扩充流水线

**依据**:竞品护城河所在。21 条 vs "适配 90% 国产软件"是当前最大差距;且特征库增长直接放大 F2/F3 两个引擎的价值。

**正当的加速方法**(行为观察,非逆向):

1. 测试机安装竞品(磨针/CCleaner/优化大师),用 **Process Monitor 记录其实际删除/移动的路径**——观察行为得到的路径是客观事实,不受版权保护;
2. 归纳为候选规则(pathPattern + 白话名 + explain + risk);
3. 走既有准入纪律:**真机验证**(删前删后目标软件功能正常、无数据丢失)后才收录 `rules.json`;
4. 每条注明验证机器与日期,存档到规则的维护记录。

**第一批候选清单**(国内装机率优先):WPS 缓存(用户已提出待验证)、百度网盘缓存、迅雷、网易云音乐缓存、爱奇艺/腾讯视频/优酷离线缓存、QQ浏览器/360浏览器 Cache、Steam shader cache、Adobe 系缓存、钉钉缓存(clean 类,与现有 migrate 类互补)。

**红线检查**:浏览器仅限 Cache 类,Cookies/Login Data/History 永不列入;聊天软件只收缓存不收消息数据;每条必须真机验证后才进库。

**验收**:特征库达到 50+ 条;每条均有真机验证记录;在 3 台不同用户真机上无一误删反馈。

### 5.3 【P2】重复文件查找(缩范围实现)

**依据**:磨针/V7 均有;典型场景"同一个安装包/视频下了三遍"。

**差异化设计——不做全盘 hash**(慢且收益低),只对**已扫描出的大文件列表**(≥100MB,≤5000 个,scan.rs 已有)做比对:先按 size 分组,同 size 组内再算快速 hash(xxHash/BLAKE3)。零新增磁盘遍历,秒级完成;重复的大文件才是空间杠杆,重复的小文件不值得用户逐个决策。

**UI**:大文件页新增「重复」筛选 chip,重复组内标出"保留 1 份可释放 X GB",删除仍走现有回收站通道与双防线(scan.rs:569-578)。

**验收**:人工构造 3 组重复大文件(含不同文件名、同内容),全部识别;比对耗时 <3s。

### 5.4 【P2】大文件页小改

- 阈值可调:100MB 默认,提供 500MB / 1GB 档位(竞品均支持自定义;实现为前端过滤即可,后端 BIG_FILE_MIN 不动);
- 排序切换:大小 ↔ 修改时间(backlog 已有);
- 单文件「移动到其他盘」延续讨论(此前已向用户提出,未排期)。

### 5.5 【P3】MFT 直读加速扫描

**依据**:WizTree 秒级全盘是品类技术标杆;本项目 35.5s 对小白够用但有感知差距。提权场景下读 NTFS MFT/USN,未提权自动回落现有并行扫描。backlog 已有,维持排期不提前——它不产生新的用户价值,只优化已达标的体验。

### 5.6 【P3】"保持 C 盘不满"(对标磨针常驻负载均衡的干净替代)

**依据**:磨针"会员期间维持 C 盘空间稳定"证明了留存价值,但其实现是常驻组件,踩本项目"无后台常驻"红线。

**干净替代**:用户显式开启后,注册 Windows 计划任务(每周/空闲时)执行「知识库增量清理 + 已迁移目录的新增数据搬运」,执行完即退出、不留进程;任务在设置页可见可关。这是"绿色承诺"与留存价值的调和方案,需求文档 §1.3 措辞届时同步细化(计划任务 ≠ 常驻,但需向用户明示)。

## 6. 坚决不做清单(负面清单)

| 不做 | 竞品案例 | 本项目对策 |

| 注销/重启硬搬系统目录 | 磨针"负载均衡"需注销配合 | 只搬知识库白名单数据目录,优雅关闭替代硬搬 |

## 7. 定位一句话
