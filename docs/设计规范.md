# C盘救星 设计规范

> 版本:v0.3(视觉与信息架构改版:清透卡片 + 侧栏导航)
> 配套文档:[需求文档.md](./需求文档.md)
> 视觉基准:[ui-redesign.html](./ui-redesign.html) 的「F · 清透卡片」——本规范所有视觉描述以该稿为唯一准绳,措辞与稿冲突时以稿为准。

## 1. 设计原则

面向小白用户,一切设计服务于三个词:

1. **看得懂**——不出现「缓存」「联接」「AppData」等术语裸奔,每个概念都有白话解释。说「微信占用的临时文件」,不说「xwechat 目录」。
2. **不害怕**——用颜色和文案明确传达安全感:能删的绿色、要注意的黄色、动不了的灰色;任何操作前讲清后果,操作后可反悔。
3. **少思考**——默认路径永远是「一个大按钮」;高级选项折叠收起,不打扰主流程。

## 2. 信息架构与页面结构

改版后由「标签页式报告页」调整为**左侧常驻导航 + 右侧内容区**(参考系统设置类工具,可信度最高):

```
┌──────────┬──────────────────────────────┐
│ [标] C盘救星│  概览 / 各功能页内容区          │
│          │                              │
│ ▸ 概览    │                              │
│   垃圾清理 │                              │
│   搬家瘦身 │                              │
│   大文件   │                              │
│   设置     │                              │
│ (插画)    │                              │
└──────────┴──────────────────────────────┘
```

- 侧栏五项常驻:**概览 / 垃圾清理(F2)/ 搬家瘦身(F3)/ 大文件(F4)/ 设置**。当前项白底 + 左侧主色竖条高亮。
- **空间地图(F1)** 不占侧栏一级位,从概览页容量卡的「查看空间分布 ›」进入(对小白是「看热闹」性质,不挡主流程)。
- **已搬家管理** 从「搬家瘦身」页内进入(查看/还原已迁移项)。
- 主流程仍只有两跳:**概览页 →(开始体检 · 扫描)→ 概览行动区亮起「可清理 X GB · 一键清理」→ 点完成**。「体检报告」的职责融入概览页的行动区与各功能页,不再是独立的一跳页面。

## 3. 页面设计规格

### 3.1 概览页(仪表盘)

侧栏「概览」对应页。自上而下三块白卡浮在清透蓝底上:

```
┌────────────────────────────────────────────┐
│  C 盘 (系统盘)                      (3D插画) │
│  容量 100 GB · 已用超过八成,建议清理          │
│  ╭─────╮   已用 84.6 GB / 100 GB · 剩余 15.4│
│  │ 85% │   ▐███▌▐██▌▐█▌▐▌░░  (分段容量条)   │
│  ╰─────╯   ■系统 ■应用 ■临时 ■其他 ░剩余     │
├────────────────────────────────────────────┤
│  (扫帚) 可清理约 7.4 GB      [ 开始体检(约1分钟)]│  ← 绿色行动区
│         体检后一键清理垃圾与搬家瘦身            │
├────────────────────────────────────────────┤
│  ⏱ 上次体检 · 3 天前 · 释放了 12.3 GB      › │
└────────────────────────────────────────────┘
```

- **容量卡**:左侧渐变环显示已用百分比(85%),右侧一行文字数字 + **分段容量条**(把「用在哪了」拆成 系统与保留 / 已装应用 / 临时·垃圾 / 其他 / 剩余,带图例与 GB 数)。环形颜色随用量:≤80% 主色蓝、80~90% 警示黄、>90% 危险红 + 呼吸动效。
- **绿色行动区**:唯一主按钮「开始体检(约 1 分钟)」置于此,配「可清理约 X GB」引导。按钮文案带预期耗时;禁止「深度清理」「极速优化」等夸大词。
- **上次体检条**:有历史时显示,建立信任;点击可进入历史/详情。
- 扫描中:环形转为进度环,行动区文案变为正在扫描的目录名(让等待可感知)。
- ⚠️ 分段容量条需后端按类别聚合统计(见需求文档 F1);若某类取数失败,降级为「已用 / 剩余」两段,不显示假数据。

### 3.2 体检结果与「一键优化」口径

体检完成后,概览行动区呈现可清理总量并提供「一键优化」;明细分散在各功能页(垃圾清理 / 搬家瘦身 / 大文件)。核心口径不变:

- 「一键优化」**不包含**迁移搬家——迁移需要退出微信等软件,不适合无感自动执行;概览与「搬家瘦身」页单独引导:「微信、企业微信还能搬走 4.2GB,去搬家 ›」。
- 「预计可释放 / 可清理」**只计垃圾 + 可搬家**(示例:3.2+4.2=7.4GB),大文件为「待你决定」、不并入总数——口径与需求文档 3.5 一致,避免双重计数和空头承诺。

### 3.3 清理明细列表(垃圾清理页)

侧栏「垃圾清理」对应页。顶部为摘要条(扫帚图标 + 「发现可清理垃圾 X GB」+ 右侧绿色「一键清理」),下方为明细白卡,每个清理项为一行:

```
┌────────────────────────────────────────────┐
│ ☑ 浏览器缓存(Edge)               1.2 GB   │
│   网页图片和文件的临时副本,删除后             │
│   首次打开网页会稍慢,之后恢复正常。            │
│   (药丸) 删了有代价                          │
└────────────────────────────────────────────┘
```

- 必备四要素:**勾选框、白话名称、大小、后果说明**。
- 风险徽标与需求文档 F2 三级对齐,改版后以**圆角药丸**呈现(浅底 + 深字):🟢 放心删(`safe`,自动勾选)/ 🟡 删了有代价(`cost`,默认勾选)/ 🟠 谨慎(`caution`,默认不勾)/ ⚪ 不能删(灰色卡片,解释原因)。
- 设计稿与演示素材**不得使用未收录条目**做示例(如小程序缓存首版未收录,见需求文档 F2 收录准入)——示例会被当成需求做进产品。
- 关联软件运行中:卡片右侧显示「请先退出微信 [帮我退出]」,勾选框禁用。

### 3.4 搬家瘦身页(F3)

- 列表结构同 3.3,每项额外显示:目标位置(如 `D:\AppDataMove\`,可在设置改)。
- 点「开始搬家」进入**向导式流程**(小白场景下最稳的交互):
  1. 「请退出微信」步骤页(检测到进程退出后自动亮起下一步)
  2. 搬家进度页(进度条 + 「正在搬 3.2GB,别关机哦(已自动帮你阻止电脑休眠)」)
  3. 完成页(「已释放 3.2GB,微信可以正常打开了,试试看」)
- 失败时:自动回滚后用绿色提示「已恢复原状,你的数据没有任何变化」——失败文案的重点是**安抚**,不是报错。

### 3.5 大文件页(F4)

- 表格列:文件名(带类型图标)、大小、修改时间、操作(打开位置 / 删除)。
- 删除进回收站,行内提示「已移到回收站(空间还没释放),清空回收站后才真正腾出来 [立即清空]」——进回收站不等于释放空间,不讲明白这层,用户删完发现首页环形图不动会认为工具无效。
- 文件超过回收站容量上限时系统会转为**永久删除**,该分支必须弹红色二次确认(「这个文件太大放不进回收站,删除后无法找回,确定吗?」+「永久删除」「先不删」)。
- 系统文件(pagefile.sys 等)显示但操作区为「这是系统文件,不能删 ⓘ」。

## 4. 视觉规范

### 4.1 整体视觉语言

**清透卡片**:浅蓝清透环境底(带角落柔光) + 白色卡片浮层 + 左侧常驻导航。气质干净、通透、有呼吸感,同时靠系统设置般的侧栏结构传达「可信、专业」。

去 AI 味纪律(全局遵守):

- **不用 emoji 当功能图标**——一律用线性描边图标(见 §4.5);emoji 仅可出现在完成庆祝等极少数文案里。
- 装饰克制:除概览页的等距 3D 插画与容量环渐变外,不滥用发光、彩色投影、氛围纹理。
- 不堆营销话术、不起风格花名。

### 4.2 色彩

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色(蓝) | `#2563EB` | 导航选中、容量环、链接、复选框选中。视觉稿取近似值 `#1A6EF5`,与主色视为同一蓝,以 tokens 为准 |
| **行动色(绿)** | `#0EAF6D` | **主行动按钮**(开始体检、一键清理)。改版后主按钮由蓝改绿,绿=可安全执行的正向操作 |
| 成功/安全 | `#16A34A` | 完成状态、成功提示 |
| 警示 | `#D97706` | C 盘 80~90%、容量条「临时·垃圾」段(`#F6921E`) |
| 危险 | `#DC2626` | C 盘 >90% 爆红、不可逆操作确认 |
| 文本主/次 | `#1F2937` / `#6B7280` | |
| 环境底 | 浅蓝渐变 `#EEF4FD → #F7FAFE` + 角落柔光 | 窗口背景;卡片浮于其上 |
| 卡片 | `#FFFFFF` | 圆角 16px,柔和阴影 `0 14px 34px -22px rgba(31,66,135,.22)` |

**安全三态徽标(产品红线,呈现为圆角药丸)**:

| 分级 | 文字色 | 药丸底色 |
|------|--------|---------|
| 放心删 `safe` | `#0B9D60` | `#E2F6EC` |
| 删了有代价 `cost` | `#C56F0A` | `#FDEEDA` |
| 谨慎 `caution` | `#EA580C` | 浅橙底 |

**容量环 / 分段条**:

- 容量环:渐变描边 `#7DB4FF → #2563EB`;>90% 整环转危险红 + 呼吸动效。
- 分段容量条配色:系统与保留 `#1B57C9` / 已装应用 `#3D8BFD` / 临时·垃圾 `#F6921E` / 其他 `#AAB3C0` / 剩余 `#E7ECF3`。

深色模式:首版不做,预留 CSS 变量。

### 4.3 字体与字号

- 字体:`system-ui`(默认微软雅黑),数字用 `tabular-nums` 保证对齐;关键数字(百分比、大小)字重可加粗至 800~900。
- 字号阶梯:大数字(容量百分比)32~36px / 页面/卡片大标题 20~26px / 卡片标题 16px / 正文·解释 14~15px / 辅助 12~13px。
- 小白用户偏中老年比例高,**正文不小于 14px**,行高 ≥1.6。

### 4.4 布局与容器

- 窗口默认 1000×640(侧栏 + 主内容区),最小 900×600,不支持缩小到破坏布局。
- **侧栏**:宽约 196px;导航项高 46px、圆角 12px;当前项白底 + 左侧 4px 主色竖条。
- **主内容区**:左右留白 20~28px;卡片之间纵向间距 12px。
- **卡片**:圆角 16px,白底,柔和大阴影(见 §4.2);内边距 24~28px。
- **按钮**:主行动按钮(绿)高 48~52px、圆角 12px,一屏只允许一个主按钮;胶囊/药丸类小标签圆角 999px。
- **复选框**:圆角 7px,选中主色填充 + 白勾;触摸区不小于 22×22px(中老年友好)。

### 4.5 图标与插画

- **功能图标**:统一线性描边风格(stroke ≈ 1.9,圆角端点),如硬盘、扫帚、箱子(搬家)、文件、齿轮。**禁用 emoji 充当功能图标**。
- **等距 3D 插画**:概览页容量卡右上、侧栏底部各一处装饰性等距插画(硬盘 + 方块),浅蓝调,仅作氛围,不承载信息。落地时可由设计师提供 PNG/SVG 素材替换视觉稿中的手绘近似图。

## 5. 文案规范

| 场景 | ✅ 推荐 | ❌ 禁止 |
|------|--------|--------|
| 描述目录 | 「企业微信的临时文件」 | 「WXWork 缓存目录」 |
| 描述迁移 | 「把微信的文件搬到 D 盘,微信照常使用」 | 「创建 NTFS junction 重定向」 |
| 失败 | 「没搬成功,已恢复原状,你的数据没有变化。可能是微信没退干净,退出后再试一次」 | 「Error: Access Denied (0x5)」 |
| 危险确认 | 「删除后,旧聊天里的图片会打不开,确定吗?」+ 明确的「删除」「先不删」双按钮 | 「确定/取消」 |
| 空间单位 | 统一 GB 保留 1 位小数;<0.1GB 用 MB | 混用 KB/字节 |

- 所有确认对话框的按钮写明动作(「删除」「先不删」),禁用「确定/取消」。
- 感叹号最多出现在庆祝场景(「释放了 12.3GB!」),错误提示不用感叹号。

## 6. 交互与状态规范

- **加载**:>300ms 的操作必须有进度反馈;>5s 的必须可取消。
- **空状态**:每个列表设计空状态文案(如大文件页:「没有找到 100MB 以上的大文件,你的 C 盘很干净」)。
- **操作反馈**:清理/迁移完成用数字说话(「释放了 X GB」),并更新概览页环形图与分段条,形成闭环。
- **防误触**:批量删除、迁移等操作按钮点击后 loading 态禁止重复点击;不可逆操作二次确认。
- **键盘/无障碍**:首版保证 Tab 焦点顺序正确、按钮可回车触发即可。

## 7. 工程规范(前端)

- 目录结构:`src/pages`(页面)/ `src/components`(通用组件)/ `src/api`(Tauri command 封装)/ `src-tauri`(Rust)。
- 前后端契约:所有 Tauri command 的入参/出参在 `src/api/types.ts` 统一定义 TS 类型,Rust 侧用 serde 对应,字段命名 camelCase。
- 知识库 JSON 放 `src-tauri/rules/`,结构:`{ id, pathPattern, filePatterns?[], displayName, explain, risk: "safe"|"cost"|"caution", needsAdmin?, relatedProcesses[], action: "clean"|"migrate"|"guide" }`;`filePatterns` 用于目录内混有不可删文件的场景(如缩略图缓存目录只删 `thumbcache_*.db`),缺省清理整个目录内容;`needsAdmin` 标记系统级清理项,普通权限下置灰(需求文档 §3.5 权限矩阵);`pathPattern` 支持 `%USERPROFILE%/%APPDATA%/%LOCALAPPDATA%/%WINDIR%/%SystemDrive%` 占位符,`special:` 前缀表示非文件路径目标(如 `special:recycle-bin` 走系统回收站接口);`guide` 用于自带官方「更改存储位置」入口的软件(优先引导官方入口,junction 兜底);`relatedProcesses` 仅作 UI 展示,占用判定以 Restart Manager 文件锁为准(见需求文档 F2)。
- 中文路径为一级测试用例:用户名含中文、目标盘路径含空格、超长路径(>260 字符)必须覆盖。
