# C盘救星 设计规范

> 版本:v0.2(与需求文档 v0.2 同步修订)
> 配套文档:[需求文档.md](./需求文档.md)

## 1. 设计原则

面向小白用户,一切设计服务于三个词:

1. **看得懂**——不出现「缓存」「联接」「AppData」等术语裸奔,每个概念都有白话解释。说「微信占用的临时文件」,不说「xwechat 目录」。
2. **不害怕**——用颜色和文案明确传达安全感:能删的绿色、要注意的黄色、动不了的灰色;任何操作前讲清后果,操作后可反悔。
3. **少思考**——默认路径永远是「一个大按钮」;高级选项折叠收起,不打扰主流程。

## 2. 信息架构与页面结构

```
首页(仪表盘)
├── 体检报告页(扫描完成后)
│   ├── Tab1 垃圾清理(F2)
│   ├── Tab2 搬家瘦身(F3)
│   └── Tab3 大文件(F4)
├── 空间地图页(F1,从报告页「查看空间分布」进入)
├── 已搬家管理页(查看/还原已迁移项)
└── 设置页(目标盘、日志、关于)
```

- 主流程只有两跳:**首页 →(扫描)→ 体检报告 → 点按钮完成**。
- F1 的目录树分析对小白是「看热闹」性质,放二级入口,不挡主流程。

## 3. 页面设计规格

### 3.1 首页(仪表盘)

```
┌─────────────────────────────────────┐
│              C盘救星                 │
│                                     │
│         ╭─────────────╮             │
│         │   C盘环形图   │   85%      │  ← 用量>90% 环形变红+呼吸动效
│         │  剩余 15.4GB  │            │
│         ╰─────────────╯             │
│                                     │
│      [ 开始体检(约1分钟)]           │  ← 唯一主按钮,占视觉中心
│                                     │
│   上次体检:释放了 12.3GB · 3天前     │  ← 有历史时显示,建立信任
└─────────────────────────────────────┘
```

- 扫描中:环形图变为进度环,下方滚动显示正在扫描的目录名(让等待可感知)。
- 文案规范:按钮文案带预期耗时;禁止「深度清理」「极速优化」等夸大词。

### 3.2 体检报告页

```
┌─────────────────────────────────────┐
│  体检完成!预计可释放 7.4 GB          │
│                                     │
│  ┌─────────┐ ┌─────────┐ ┌───────┐  │
│  │ 垃圾文件 │ │ 可搬家   │ │ 大文件 │  │
│  │ 3.2 GB  │ │ 4.2 GB  │ │ 1.2GB │  │
│  └─────────┘ └─────────┘ └───────┘  │
│                                     │
│      [ 一键优化(推荐)]              │  ← 执行:全部「放心删」+ 默认勾选项
│      查看明细,自己决定 ›             │  ← 半懂用户入口,进 Tab 页
└─────────────────────────────────────┘
```

- 「一键优化」**不包含**迁移搬家——迁移需要退出微信等软件,不适合无感自动执行;报告页单独引导:「微信、企业微信还能搬走 4.2GB,去搬家 ›」。
- 「预计可释放」**只计垃圾 + 可搬家**(示例:3.2+4.2=7.4GB),大文件卡片副文案为「待你决定」、不并入总数——口径与需求文档 3.5 一致,避免双重计数和空头承诺。

### 3.3 清理明细列表(Tab1)

每个清理项为一行卡片:

```
┌─────────────────────────────────────┐
│ ☑ 浏览器缓存(Edge)          1.2 GB  │
│   网页图片和文件的临时副本,删除后       │
│   首次打开网页会稍慢,之后恢复正常。      │
│   🟡 删了有代价                       │
└─────────────────────────────────────┘
```

- 必备四要素:**勾选框、白话名称、大小、后果说明**。
- 风险徽标与需求文档 F2 三级对齐:🟢 放心删(`safe`,自动勾选)/ 🟡 删了有代价(`cost`,默认勾选)/ 🟠 谨慎(`caution`,默认不勾)/ ⚪ 不能删(灰色卡片,解释原因)。
- 设计稿与演示素材**不得使用未收录条目**做示例(如小程序缓存首版未收录,见需求文档 F2 收录准入)——示例会被当成需求做进产品。
- 关联软件运行中:卡片右侧显示「请先退出微信 [帮我退出]」,勾选框禁用。

### 3.4 搬家瘦身(Tab2)

- 列表结构同 3.3,每项额外显示:目标位置(如 `D:\AppDataMove\`,可在设置改)。
- 点「开始搬家」进入**向导式流程**(小白场景下最稳的交互):
  1. 「请退出微信」步骤页(检测到进程退出后自动亮起下一步)
  2. 搬家进度页(进度条 + 「正在搬 3.2GB,别关机哦(已自动帮你阻止电脑休眠)」)
  3. 完成页(「已释放 3.2GB,微信可以正常打开了,试试看」)
- 失败时:自动回滚后用绿色提示「已恢复原状,你的数据没有任何变化」——失败文案的重点是**安抚**,不是报错。

### 3.5 大文件(Tab3)

- 表格列:文件名(带类型图标)、大小、修改时间、操作(打开位置 / 删除)。
- 删除进回收站,行内提示「已移到回收站(空间还没释放),清空回收站后才真正腾出来 [立即清空]」——进回收站不等于释放空间,不讲明白这层,用户删完发现首页环形图不动会认为工具无效。
- 文件超过回收站容量上限时系统会转为**永久删除**,该分支必须弹红色二次确认(「这个文件太大放不进回收站,删除后无法找回,确定吗?」+「永久删除」「先不删」)。
- 系统文件(pagefile.sys 等)显示但操作区为「这是系统文件,不能删 ⓘ」。

## 4. 视觉规范

### 4.1 色彩

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色 | `#2563EB` | 蓝色,传达工具感和可靠 |
| 成功/安全 | `#16A34A` | 「放心删」徽标、完成状态 |
| 警示 | `#D97706` | 「有代价」徽标、C 盘 80~90% |
| 危险 | `#DC2626` | C 盘 >90% 爆红、不可逆操作确认 |
| 文本主/次 | `#1F2937` / `#6B7280` | |
| 背景/卡片 | `#F9FAFB` / `#FFFFFF` | 卡片圆角 12px,阴影 `0 1px 3px rgba(0,0,0,.08)` |

深色模式:首版不做,预留 CSS 变量。

### 4.2 字体与字号

- 字体:`system-ui`(默认微软雅黑),数字用 `tabular-nums` 保证对齐。
- 字号阶梯:大数字(剩余空间)32px / 页面标题 20px / 卡片标题 16px / 正文·解释 14px / 辅助 12px。
- 小白用户偏中老年比例高,**正文不小于 14px**,行高 ≥1.6。

### 4.3 布局

- 窗口默认 960×640,最小 800×600,不支持缩小到破坏布局。
- 栅格:内容区左右留白 24px,卡片间距 12px。
- 主按钮:高 48px、全宽或 ≥240px,一屏只允许一个主按钮。

## 5. 文案规范

| 场景 | ✅ 推荐 | ❌ 禁止 |
|------|--------|--------|
| 描述目录 | 「企业微信的临时文件」 | 「WXWork 缓存目录」 |
| 描述迁移 | 「把微信的文件搬到 D 盘,微信照常使用」 | 「创建 NTFS junction 重定向」 |
| 失败 | 「没搬成功,已恢复原状,你的数据没有变化。可能是微信没退干净,退出后再试一次」 | 「Error: Access Denied (0x5)」 |
| 危险确认 | 「删除后,旧聊天里的图片会打不开,确定吗?」+ 明确的「删除」「先不删」双按钮 | 「确定/取消」 |
| 空间单位 | 统一 GB 保留 1 位小数;<0.1GB 用 MB | 混用 KB/字节 |

- 所有确认对话框的按钮写明动作(「删除」「先不删」),禁用「确定/取消」。
- 感叹号最多出现在庆祝场景(「释放了 12.3GB!」),错误提示不用感叹号。

## 6. 交互与状态规范

- **加载**:>300ms 的操作必须有进度反馈;>5s 的必须可取消。
- **空状态**:每个列表设计空状态文案(如大文件页:「没有找到 100MB 以上的大文件,你的 C 盘很干净」)。
- **操作反馈**:清理/迁移完成用数字说话(「释放了 X GB」),并更新首页环形图,形成闭环。
- **防误触**:批量删除、迁移等操作按钮点击后 loading 态禁止重复点击;不可逆操作二次确认。
- **键盘/无障碍**:首版保证 Tab 焦点顺序正确、按钮可回车触发即可。

## 7. 工程规范(前端)

- 目录结构:`src/pages`(页面)/ `src/components`(通用组件)/ `src/api`(Tauri command 封装)/ `src-tauri`(Rust)。
- 前后端契约:所有 Tauri command 的入参/出参在 `src/api/types.ts` 统一定义 TS 类型,Rust 侧用 serde 对应,字段命名 camelCase。
- 知识库 JSON 放 `src-tauri/rules/`,结构:`{ id, pathPattern, displayName, explain, risk: "safe"|"cost"|"caution", relatedProcesses[], action: "clean"|"migrate"|"guide" }`;`guide` 用于自带官方「更改存储位置」入口的软件(优先引导官方入口,junction 兜底);`relatedProcesses` 仅作 UI 展示,占用判定以 Restart Manager 文件锁为准(见需求文档 F2)。
- 中文路径为一级测试用例:用户名含中文、目标盘路径含空格、超长路径(>260 字符)必须覆盖。
