# C盘救星(C-Disk Rescue)需求文档

> 版本:v0.2(按系统评审意见修订:补权限矩阵、reparse point 删除红线、迁移时序改 copy→校验→建链、目标盘文件系统校验等)
> 日期:2026-07-11
> 状态:待评审

## 1. 背景与目标

### 1.1 背景

Windows 用户(尤其是笔记本用户)的 C 盘普遍偏小(100~256GB),而大量软件默认把缓存、聊天数据、更新包写入 `C:\Users\<用户名>\AppData`,日积月累导致 C 盘爆红。普通用户面临三个困境:

1. **不知道空间去哪了**——资源管理器看不出哪个目录占了多少;
2. **不敢删**——AppData 里的目录名字看不懂,怕删了软件坏掉;
3. **不会迁**——微信/企业微信等软件的数据可以搬到其他盘,但 junction(目录联接)等手段超出普通用户能力。

市面上的清理工具(电脑管家类)存在捆绑推广、清理项不透明的问题;专业工具(WizTree/SpaceSniffer)只做分析不做处置,且面向专业用户。

### 1.2 产品目标

做一个**免费、无捆绑、傻瓜式**的 C 盘空间治理工具,让完全不懂电脑的用户也能安全地完成「分析 → 清理 → 迁移」全流程,把 C 盘空间要回来。

### 1.3 核心原则(产品红线)

- **绝不误删用户数据**:宁可少清 10GB,不能删错 1 个文件。所有清理项必须来自内置知识库的白名单,未知目录只展示、不提供删除按钮。
- **删除遍历绝不穿透 reparse point**:递归删除时遇到 junction/symlink 只删链接本身、永不跟入目标——防止经由联接误删其他盘的数据(本工具自己就会创建 junction,此风险是自增强的)。
- **绝不触碰浏览器隐私数据**:浏览器清理仅限 Cache 类目录;Cookies、Login Data(保存的密码)、History 永不列入清理范围——删掉用户保存的密码比少清 2GB 严重得多。
- **所有操作可回滚**:迁移失败自动回滚;清理前展示明细。
- **无广告、无捆绑、无后台常驻**:用完即走,不注册开机启动、不装驱动、不弹窗。

## 2. 用户画像

| 画像 | 描述 | 典型诉求 |
|------|------|----------|
| 主要:小白用户 | 不知道 AppData 是什么,C 盘红了只会重启或找人帮忙 | 「一键搞定,别问我问题」 |
| 次要:半懂用户 | 会用资源管理器删文件,但分不清哪些能删 | 「告诉我每一项是什么,我自己勾」 |

交互设计以小白用户为准:**默认全自动推荐方案,一个大按钮执行**;半懂用户可展开明细自行调整勾选。

## 3. 功能需求

### 3.1 F1 扫描分析(哪些目录占空间)

| 项 | 说明 |
|----|------|
| 描述 | 扫描 C 盘,可视化展示空间分布,让用户一眼看到「空间去哪了」 |
| 扫描范围 | 默认扫描:用户目录(AppData、下载、文档、桌面)、`C:\Windows` 可清理区(更新缓存等)、`C:\Program Files*`;可选全盘 |
| 展示形式 | 按目录树 + 占比条形图分层展示;每个目录标注大小、文件数;知识库命中的目录显示「这是什么」的白话解释(如:`AppData\Roaming\Tencent\WXWork` → 「企业微信的缓存和运行组件」) |
| 性能要求 | 分层扫描:先扫知识库热点目录,数秒内先出体检报告,全盘目录树后台补全。普通权限用多线程遍历(jwalk),SSD 上 100GB 盘全量 ≤ 60 秒(HDD 不作承诺);已提权时可切换 MFT 读取加速(读 MFT 需管理员打开裸卷句柄,普通权限不可用,见 3.5 权限矩阵);扫描过程显示进度 |
| 特殊处理 | ① 已是 junction/symlink 的目录标注「已迁移」并显示指向位置,不重复计入 C 盘占用;② OneDrive 按需文件:统计用实际占用(AllocationSize)口径,扫描只读元数据、绝不打开文件内容——否则会触发 OneDrive 全量下载,反而占满 C 盘;③ `WinSxS` 等硬链接密集区按去重后大小展示,避免夸大「可释放」预期 |

### 3.2 F2 安全清理(垃圾一键删)

| 项 | 说明 |
|----|------|
| 描述 | 基于内置知识库识别可安全删除的垃圾,一键清理 |
| 清理项知识库(首版) | ① 系统:用户/系统 Temp、`SoftwareDistribution\Download`(更新残留,删除前先停止 wuauserv 服务,否则文件被占用或删完立即重新下载)、`Windows.old`(功能更新残留,动辄 10~30GB,通过系统清理接口触发而非自行删除——其权限归 TrustedInstaller;风险级「可删但有代价:无法回退旧版本系统」)、传递优化缓存(Delivery Optimization)、回收站、缩略图缓存、错误转储(`.dmp`);② 浏览器:Chrome/Edge 仅限 Cache 目录(Cookies/密码/历史是红线,见 1.3);③ 开发:npm/yarn/pnpm/pip 缓存、`TypeScript` 缓存;④ WPS 缓存 |
| 分级 | 每个清理项标注风险级:**放心删**(纯垃圾,自动勾选)/ **可删但有代价**(如浏览器缓存,删后首次打开变慢,默认勾选)/ **谨慎**(如小程序缓存,删后重新下载,默认不勾选) |
| ⚠️ 收录准入(教训) | 任何第三方软件的「缓存类」目录,必须在真机上验证「删除后软件仍能正常启动」才能收录。已被证否的案例:腾讯视频 QQLive 的小游戏目录(WxMiniGame 等)删除后导致**腾讯视频无法启动**,严禁收录为清理项;微信 `xwechat\update`、小程序 Applet 缓存未经删除验证,首版**不收录**,待验证后再以「谨慎」级加入。对这类目录,产品策略是引导走 F3 迁移(整目录搬走建联接,软件无感知),而非删除 |
| 交互 | 小白路径:首页「一键清理」按钮 → 只执行「放心删」+「默认勾选」级;明细路径:展开列表逐项勾选,每项有白话解释 |
| 前置检查 | 占用判定以**文件锁**为准:用 Restart Manager API(`RmGetList`)检测哪些进程正锁定目标目录,不依赖进程名匹配(软件跨版本会改进程名,微信 3.x/4.x 连目录都换了;知识库的进程名仅作 UI 展示);占用时该项置灰并提示「请先退出 xx」;「帮我关闭」只走优雅关闭(发送关闭消息并等待退出),**绝不强杀**——强杀正在写数据库的进程会损坏用户数据,违背产品红线 |
| 执行与反馈 | 删除遍历遇 reparse point 只删链接本身、永不跟入(红线,见 1.3);显示清理进度;完成后展示「本次释放 X GB」;清理日志落盘可查 |

### 3.3 F3 迁移搬家(junction 迁到其他盘)

| 项 | 说明 |
|----|------|
| 描述 | 把 AppData 中的大数据目录移动到其他盘,原位置建 junction 联接,软件无感知。**这是与其他清理工具差异化的核心功能** |
| 可迁移目录知识库(首版) | 微信(`xwechat`/`WeChat`)、企业微信(`WXWork`)、微信开发者工具、钉钉、QQ(NT 版)、夸克/百度网盘缓存目录、Volta/nvm 的 Node 版本目录。软件自带官方「更改存储位置」入口的(如微信的文件管理设置),知识库标记为 `guide` 类型,优先引导用户走官方入口(比 junction 更抗版本更新),junction 作兜底——每个软件需逐个验证后标注 |
| 目标盘选择 | 自动推荐剩余空间最大的本地盘(排除 U 盘/移动硬盘,防止拔盘后软件异常);**校验目标分区为 NTFS**——FAT32 有单文件 4GB 上限(微信数据库文件会复制失败),exFAT 无 ACL,均拒绝;迁移前校验目标盘剩余空间充足;统一迁入 `<盘>:\AppDataMove\` |
| 执行流程(copy→校验→换名→建链,全程源数据完整存在) | ① 以 Restart Manager 确认无进程锁定,提示用户退出(白名单进程可代为优雅关闭);② 记录源目录文件数与总字节数;③ **复制**(而非移动)到目标盘,保留时间戳与 NTFS 备用数据流(ADS);④ 双校验:文件数 + 总字节数一致;⑤ 源目录改名为 `<名>.bak`;⑥ 原位置建 junction;⑦ 引导用户启动软件确认正常;⑧ 确认后才删除 `.bak`(或保留 N 天后自动清理)。任一步失败按逆序回退——因源数据全程未动,回滚只是删副本/改回名,不存在「搬回时再出错」的二次风险。跨盘 Move 本质是复制+删除且非原子,中途断电会把数据劈成两半,故弃用 v0.1 的 Move 方案。迁移期间调用 `SetThreadExecutionState` 阻止系统睡眠(笔记本合盖比手动关机常见得多) |
| 还原功能 | 已迁移列表中每项提供「搬回 C 盘」按钮,用于软件更新异常时自救;还原复用与迁移相同的事务化流程(copy→校验→删联接→就位),且动手前**校验 C 盘剩余空间**——典型事故:迁走 20GB 后 C 盘又被填满,还原中途空间不足数据劈两半 |
| 风险提示 | 迁移前用白话告知:「软件大版本更新时极小概率出问题,出问题可以在本工具里一键搬回」 |
| 边界 | **白名单优先于启发式**:知识库白名单内的目录即使含 exe/dll 也允许迁移(junction 对路径解析完全透明,`WXWork` 这类含运行组件的目录经真机验证后照常收录);「含 `.exe` 不迁」的启发式仅用于未来开放用户自选目录的场景。不允许迁移:系统目录、`Program Files` 下的程序主安装目录、已是联接的目录、目标为网络盘/可移动盘/非 NTFS 分区 |

### 3.4 F4 大文件排查(全盘找大文件)

| 项 | 说明 |
|----|------|
| 描述 | 按大小倒序列出 C 盘大文件(默认 >100MB),帮用户发现遗忘的下载、视频、安装包 |
| 展示 | 文件名、大小、路径、修改时间、类型图标;按类型筛选(视频/压缩包/安装包/镜像) |
| 操作 | 「打开所在位置」「删除(进回收站)」;系统关键文件(`pagefile.sys`、`hiberfil.sys`、Windows 目录内文件)只展示不可删,并附白话解释 |
| 安全 | 删除一律进回收站,保留反悔余地;但**进回收站 ≠ 释放空间**,删除后明确提示「清空回收站后才真正腾出」并提供一键清空入口(否则用户删完发现环形图不动,信任感崩塌);超过回收站容量上限的超大文件,系统会转为**永久删除**,该分支必须红色二次确认;大文件列表对 F2/F3 已覆盖的路径去重,避免同一空间双重计数 |

### 3.5 通用需求

- **首页仪表盘**:C 盘用量环形图(爆红状态用醒目颜色)+「预计可释放 X GB」+ 一个主按钮「开始体检」。
- **体检报告**:扫描后汇总三类结果。「预计可释放」**只计入可清理垃圾 + 可迁移**(这两类是工具可直接兑现的);大文件属「待用户人工决定」性质,单独展示、不并入总数——三类数字直接相加会双重计数(可迁移目录里的大视频会同时出现在大文件列表)且承诺了工具兑现不了的空间。
- **操作日志**:所有删除/迁移操作记录到本地日志文件,出问题可追溯。
- **权限矩阵**:默认以普通权限运行;需提权的操作单独触发 UAC 并说明原因,提权通过独立辅助进程实现(已运行的进程无法自我提权)。各功能在两种权限下的行为:

| 功能 | 普通权限 | 管理员 | 拒绝提权后的降级体验 |
|------|---------|--------|---------------------|
| F1 扫描 | 多线程遍历,部分系统目录/其他用户目录不可见 | 可选 MFT 加速,目录全可见 | 报告标注「部分系统区域未扫描」 |
| F2 用户级清理(Temp/浏览器/开发缓存) | ✅ | ✅ | 不受影响 |
| F2 系统级清理(更新缓存/Windows.old) | ❌ | ✅ | 条目置灰,说明「需要管理员权限」 |
| F3 迁移(建 junction 无需管理员) | ✅ | ✅ | 不受影响 |
| F4 大文件 | 用户可访问范围 | 全盘 | 系统目录内只展示 |

## 4. 非功能需求

| 维度 | 要求 |
|------|------|
| 安装包体积 | ≤ 15MB(Tauri 优势,与 Electron 方案 100MB+ 形成对比);安装器需检测 WebView2 Runtime,缺失时引导在线安装——Win11 自带,老旧 Win10 存在缺失/损坏个案,而这恰是本产品的目标用户群,此场景必须接住 |
| 内存占用 | 空闲时:主进程 ≤ 50MB;含 WebView2 全部子进程 ≤ 250MB。WebView2 是多进程架构,仅渲染进程就常年 100MB 级,原「整体 ≤ 100MB」不可达成,验收按本口径执行 |
| 系统兼容 | Windows 10 1809+ / Windows 11,x64;最低小版本以 Tauri 2.x 官方支持范围为准,开发启动前核对官方文档钉死 |
| 隐私 | 运行期完全离线,不上传任何数据;无遥测(或仅匿名崩溃上报且默认关闭)。唯一例外:安装环节 WebView2 缺失时需联网获取,在安装器界面明示 |
| 中文路径 | 全链路正确处理中文用户名/路径(开发中已踩过 GBK/UTF-8 坑);**超长路径(>260 字符)**与中文路径同列专项测试——npm 缓存、node_modules 嵌套是重灾区 |
| 更新 | 知识库(清理项/可迁移项规则)以 JSON 配置内置,支持随版本更新;首版不做在线热更新 |

## 5. 技术选型

| 层 | 选型 | 理由 |
|----|------|------|
| 框架 | Tauri 2.x | 包体小、内存低,契合「给 C 盘不够的用户」的定位;界面用团队熟悉的 Web 技术 |
| 前端 | Vue 3 + TypeScript + Vite | 团队现有技能栈 |
| 后端(Rust) | 扫描:`jwalk`(并行遍历,普通权限)+ MFT 方案(提权后可选加速);junction:Windows API(`CreateFile`+reparse point)或 `junction` crate;文件占用检测:Restart Manager API(`RmStartSession`/`RmGetList`,按文件锁判定而非进程名);进程枚举展示:`sysinfo` crate;提权操作:独立提权辅助进程承载 | 文件系统重操作放 Rust 侧保证性能 |
| 知识库 | 内置 JSON 规则文件(路径模式、白话解释、风险级、关联进程名) | 规则与代码分离,便于维护和社区贡献 |

## 6. 里程碑

| 阶段 | 内容 | 验收标准 |
|------|------|----------|
| M1 骨架 | Tauri 项目搭建、首页仪表盘、F1 扫描分析 | 能扫出目录树和大小,60 秒内完成 |
| M2 清理 | F2 安全清理 + 知识库首版 | 在测试机上清理后所有关联软件正常启动 |
| M3 迁移 | F3 junction 迁移 + 回滚 + 还原 | 微信/企业微信迁移后正常使用;断电/占用场景下回滚成功;**迁移后触发软件自动更新,验证 junction 存活**或还原路径可用;还原在 C 盘空间紧张场景下正确拒绝并提示 |
| M4 打磨 | F4 大文件 + 体检报告 + 一键优化 | 小白用户(找 3 位非技术亲友)不看说明能独立完成一次优化 |

## 7. 风险与对策

| 风险 | 对策 |
|------|------|
| 知识库规则错误导致误删 | 规则三级评审;「放心删」级只收录有官方依据或大规模验证的路径;灰度:新规则先以「谨慎」级上线 |
| 软件更新后 junction 失效/异常 | 提供一键还原;知识库维护「已知不兼容版本」黑名单 |
| 被杀毒软件误报(改 AppData、建联接、触碰微信数据目录,均是窃密类恶意软件的典型行为特征) | 代码签名;主动向 360/腾讯电脑管家/火绒提交白名单;首版先在小范围分发收集误报情况 |
| 用户中途强制关机导致迁移中断 | 迁移前写入事务日志,启动时检测未完成事务并提示恢复 |
